# 更新日志

记录优先写用户能感知到的变化；过于内部的实现细节只在确实有助于排障时保留。

## 5.7.0

- 总览页新增“配置引导”入口：点击后在大弹窗内完成问卷式首次配置，覆盖基础连通、目标 QQ 与平台、运行策略与权限、快捷模型配置、人格与世界知识、主动能力、日程初始化、私聊主动强度、群聊唤醒/插话、关系网和主动消息测试。
- 首次配置引导已落地保存：完成前会把当前草稿写入插件配置与运行态，主动消息测试前也会先保存目标用户、平台和模型等必要配置，避免刚填完 QQ 就测试不到目标用户。
- 快捷模型配置改为在弹窗内直接选择并逐项测试四类模型；所有必需模型测试通过后才能进入下一步，避免用户只选模型但实际不可用。
- 人格与世界知识步骤改为先选择人格，再手动点击推导世界知识草稿；推导完成后可在弹窗内修改确认，并明确提示世界知识只服务插件，实际对话人格仍由 AstrBot 配置决定。
- 日程与观察步骤新增“生成今日日程和当前细化”按钮，并在弹窗内展示生成结果；私聊主动强度预设会实时展示并允许手动调整对应参数。
- 群聊与关系网拆分为独立步骤；唤醒增强默认按现有配置关闭，关闭时不展示其下配置；关系网首个用户词条配置去掉不适合首次引导的绑定 QQ、重要记忆草稿和优先级。
- 主动消息测试复用排障链路：点击后等待约 15 秒进行完整主动消息测试，实时展示结果；通过后可保存完成并选择自行探索或继续后续进阶引导。
- 配置引导补齐步骤阻塞校验和进度条状态，已完成步骤显示绿色；移除页面切换过渡动画，减少弹窗内操作时的跳动感。
- 修正 OpenAI 兼容 Provider 在插件下拉框里统一显示为 `openai` 的问题：现在优先使用 AstrBot 供应商显示名/Provider ID，必要时再按 API 地址或模型名推断火山引擎、DeepSeek、百炼等供应商。
- 修正主动候选总览把已拦截候选也算进“未发送候选”、以及重复合并次数无限增长导致来源分布显示上万次的问题；候选合并计数现在会自动封顶，旧数据会在启动/保存时压缩，候选总数改为展示记录数，触发次数只作为说明。
- TTS 生成前的空内容判断进一步收紧：只有情绪标签、动作括号或纯标点时不再调用语音合成；带正文的情绪标签仍会保留给支持的 TTS 供应商。
- 移除“主链会话锁兼容模式”功能和配置项；新版本不再提供该开关，也不再额外创建同会话主链锁。
- 修复群聊主动插话可能把 `（空字符串）`、`【不回复】`、`null` 等“不发言占位”当作正文发送并写入 `last_bot_interjection` 的问题；源头已从“正文/空文本”改为 JSON 决策 `should_reply + text`，插话清洗和群聊发送前静默拦截也会兜底归一化括号、引号和结尾标点。
- 修正私聊图片链路的上下文优先级：单图回复仍避免把旧邀约/旧日程当当前图片内容，但最近对话里用户明确规定“这张/下一张图片只回某句、不要回复其他内容”时会优先遵守，不再被图片边界提示降权成普通语气背景。
- 修复休息静默闸门把“你不用回复图片内容，夸夸我就行”误判成“不要回复我”的问题；带有图片/内容等明确作用域和替代回复方式的句子不再触发勿扰，已有误判静默也会在读取时自动清理。

## 5.6.15

- 移除“私聊工具安全裁剪”功能和相关配置项，不再在普通私聊被动主链前改写 AstrBot 工具列表。
- 修复配置备用生图 API 后，`/陪伴 生成穿搭` 在部分适配器上可能把同一张穿搭图发送两次的问题；穿搭命令现在走单一媒体回复链路，并对同一会话同图同文增加短时间去重。

## 5.6.14

- 修复私聊单图/GIF 走手动分段发送时，`[[PCTTS:...]]` 私有 TTS 保护占位可能出现在后续分段或日志预览里的问题；现在发送前会先尝试按当前事件还原为 `<tts>` 语音块，无法还原的孤儿占位会清理掉，分段链路也有兜底过滤。

## 5.6.13

- 修复自然语言生图/改图可能把“我会牢牢记住你”的上下文注入块、内部 instruction 或记忆分区说明当作图片 prompt 传给生图后端，导致生成图片里出现大段提示词/记忆文本的问题；现在图片模型只接收清洗后的视觉连续性参考（穿搭、地点、外观、颜色、背景等），并明确禁止生成文字说明栏。
- 每日穿搭图的记忆连续性参考也改用同一套视觉清洗，避免 MemoryCompanion 标签、内部分工说明或非视觉记忆进入图片 prompt。

## 5.6.12

- 修复私聊单图识图链路过度依赖配置顺序的问题：会话配置里的空图片转文字字段不再遮住 AstrBot 全局“默认图片转文字模型”；成功识图的视觉模型会被记录为近期恢复候选，但不会覆盖 AstrBot 明确配置的首位。首选不可用或失败降权后，才会用最近成功模型加速恢复。识图模型返回空摘要或“看不到图”类不可用摘要时不再提前结束，而是临时降权并继续尝试下一个候选。排障页也会显示私聊图片识别的最近成功模型、当前候选可用性和临时降权原因。
- 收紧私聊图片识别的快速配置边界：视觉转述候选只允许 AstrBot 默认图片转文字模型、插件识图模型和已验证的视觉恢复候选，不再把快速配置里的复杂推理模型/插件主模型或 AstrBot 默认聊天模型拿来跑识图；历史运行态中记录过的 `plugin_main` 视觉成功也会被过滤，避免旧状态绕回。
- 调整拓展页模型配置页：模型分组不再默认折叠，也不再通过鼠标移入/点击标题展开收起；快速配置/精准配置仍只负责显示当前模式需要填写的模型项。
- 私聊被动主链的高风险工具裁剪改为可选功能“私聊工具安全裁剪”，默认关闭；需要更强安全边界时可在拓展页“功能开关 -> 通用能力”或官方配置页开启，开启后才会裁掉 Shell、文件读写、启动程序和跨会话发消息等工具。
- 强化私聊单图无视觉摘要保护：识图失败且图片没有直挂给主模型时，不再进入主链生成“看图回复”，改用更自然的短句说明没读出图片、请用户点一下重点，避免主动状态/历史话题里的“路上、天色、冰激凌”等旧上下文被模型当成图片内容并分段漏出。
- 修复私聊单图识图回复后“下一句不记得刚才看过图”的问题：手动发送链路现在会把“用户发图/图片摘要 + 已发出的 Bot 回复首段或单段回复”写成一轮标准 AstrBot 会话历史，并同步给 MemoryCompanion 的可见对话窗口；保留轮数继续按 AstrBot/记忆插件自身上下文配置裁剪，不再用固定一次的临时缓存。
- 修复私聊单图识图失败后仍进入主链导致模型凭历史猜图的问题：没有可靠视觉摘要且图片未直接交给支持图片的主模型时，会丢弃看图式主链回复，改为短句请用户补充要看的位置。
- 收窄群聊关系网“近期参与者”注入：主人/目标陪伴用户不会再仅因最近发过言就被注入到其他群友当前发言里，避免普通群友被误套主人身份。
- 吸收 PR #45 的运行时保护：通用文本清理会移除泄漏的 `<think>` / `<reasoning>` 块；TTS 生成前会跳过纯标点朗读文本；群黑话释义遇到模型返回非 JSON 时只跳过本轮刷新，不再进入 30 分钟短冷却。
- 优化主动强度预设梯度：`标准偏主动 / 私聊高频 / 群聊活跃 / 在线陪伴` 的有效差异显著拉开；最高档“在线陪伴”不按每日主动次数或每日群聊插话次数封顶，空闲 0 分钟、最小间隔 5 分钟、群唤醒 3 秒冷却，并忽略 Token 软限额降载，但仍保留免打扰、休息、用户拒绝、隐私和每日 Token 硬限额。
- 修复主动消息发送失败后复用待重发内容时，可能把“直接在当前对话中输出这条主动消息”这类内部发送指令当作正文发给用户的问题；现在新生成、复核改写、发送前硬拦截和待重发缓存都会识别并丢弃这类提示词泄漏，旧污染缓存也会自动清空并重新排程。
- 收束主动消息发送闸：新增统一本地校验入口，主动复核、真实发送前、待重发读取和待重发写入都复用同一套“回执/内部指令/工具摘要/空内容”判定；待重发内容如果只能清出安全正文会改写后继续，否则直接丢弃并重新排程。
- 优化 QQ 空间 Cookie 自动获取：参考 `astrbot_plugin_qzone_ultra` 的绑定思路，自动模式会尝试 `get_cookies/get_credentials`、多种 OneBot action 调用签名、多域名候选和嵌套/JSON 返回结构；缺少 `uin` 时会用 `get_login_info` 补齐，并在扩展页状态中记录上次 Cookie 获取是否拿到 `uin/skey/p_skey`。

## 5.6.11

- 修复主动搜索使用自定义 GET 接口时，配置为 API 尾插或尾部 `?`/`&` 的地址会拼接失败的问题；现在支持 `{query}`、`{{query}}`、`${query}` 占位、已有查询参数和尾插写法，超时提示也改成更直观的“请求超时”。
- 修复外部图片结果下载遇到鉴权/防盗链时容易 403 的问题：主动图片结果拉取会使用浏览器式请求头，并在同源结果地址上安全携带自定义鉴权头；拓展页也补齐主/备用图片 API 自定义头配置。
- 修复用户引用的消息本身又引用更早消息时，回复前只能看到一层引用的问题；现在会最多读取 3 层引用链，并把原始被引用内容注入 `reply.chain` 上下文，同时引用图片识别也会沿链查找。
- 优化朋友用户长时间未回应后的主动念头降级：不再只是拉长发送间隔，而会按未回应程度逐级改成低压文字、留出空间，并清理旧念头语义缓存，避免旧高压念头隔很久后原样回来。
- 修复“习惯关心”把用户常说的“早”当作普通主动话题，导致早上已经自然聊过后仍发早安类主动的问题；带称呼的“比折，早……”也会按早间问候去重和拦截。

## 5.6.10

- 修复 `<bubble/>` 伪分段标记泄漏到聊天的问题：内部历史、表达学习、主动/工具直发和发送前兜底都会清理未知自闭合控制标签；同时清理本地 `companions.json` 中已污染的 `<bubble/>` 样本并保留备份。
- 修复 QQ 空间发布成功后 Bot 自己“不记得刚才发了什么说说”的问题：说说发布成功会写入本插件自我时间线，并在可用时同步到 MemoryCompanion 的人格生活记忆；用户问“刚才发了什么说说/QQ 空间发了什么”时会按真实发布记录回答。
- 修复拓展页在部分 AstrBot Page 桥实现下只显示“加载失败：未找到该路由”的问题：前端请求会自动尝试 `page/...`、裸路由和完整插件名前缀等多种端点格式，命中后缓存可用格式。
- 修复撤回图片缓存持续膨胀的问题：`recall_message_images` 现在会跟随撤回缓存秒数自动清理过期图片，并新增 `recall_message_image_cache_max_mb` 磁盘体积上限（默认 256MB）；启动维护也会执行一次清理，避免群聊图片/GIF 长期堆到数 GB。
- 修复 QQ 状态同步在 NapCat 等协议端上的兼容风险：基础在线/忙碌状态仍默认可用，但“QQ 自定义短状态”拆成独立开关并默认关闭；执行层不再调用已知会让部分 NapCat 版本断连的 `set_custom_online_status`，睡觉/忙碌等状态在未开启自定义短状态时会回退为标准在线/忙碌。
- 主动搜索新增可选自定义搜索接口：可在拓展页“功能开关 -> 长线主动 -> 主动搜索详情 -> 自定义搜索接口”或官方配置页“主动搜索”分组填写接口地址、API Key 和模型；留空仍使用 AstrBot 全局网页搜索，且只接管主动搜索空档探索，不影响新闻阅读和群黑话联网参考。
- 修复缺少 `torch` 等可选模型依赖时，私聊图片存在性检测可能把 `on_private_message` 整个处理器炸掉的问题：私聊空消息判断、图文分支和自然语言生图参考图检测统一改用安全图片检测，缺依赖时按无图片继续并写入降级日志。
- 增加 AstrBot 指令注册诊断日志：插件初始化时会输出 `/陪伴`、`/陪伴群` 及其别名是否已进入 AstrBot handler registry，方便区分“指令未注册”和“指令查询/LLM 执行器缓存未刷新”。
- 补充 QQ 空间评论回复排障确认：评论收件箱自动回复成功时日志会记录说说 ID、评论 ID、评论者和实际回复文本；当前不会默认打印对方原评论全文，避免公开评论内容在日志中过度展开。
- 修复主动私聊发送的核心兜底平台寻址：当历史 UMO 为 `aiocqhttp:FriendMessage:...` 但当前平台实例 ID 是 `default` 等其他名称时，兜底发送会使用实际平台实例 ID 重建 `MessageSession`，避免 AstrBot 核心日志继续报 `cannot find platform for session aiocqhttp:...`；平台实例本身就是 `aiocqhttp` 的环境会保持原 ID，不写死为 `default`。
- 修复群聊消息误影响私聊主动的问题：私聊主动的空闲判定、问候候选、发送前 recent-chat guard，以及发送期间“用户有新消息则取消”保护，都改为只看私聊专用活动时间和私聊入站计数，群聊发言不再打断私聊主动消息或排障页主动消息测试。
- 新增低概率“群里冒泡但私聊未回”的私聊候选：当用户长时间未回复上一条私聊主动、但本人又在群聊里发言时，会在冷却和人格倾向都合适的情况下低概率排入一条低压抱怨/撒娇式私聊候选；不会直接硬发，也不会因为普通群聊消息打断既有私聊主动。
- 强化 QQ 空间自拍配图参考图链路：配图提示词设计前会预检自拍参考图状态，选择 `selfie` 时会显式把今日穿搭/人设参考图传给生图入口，并在日志和扩展页摘要状态中记录参考图路径与是否存在，便于排查“自拍不像参考图”的问题。
- 优化 QQ 空间配图提示词逻辑：配图模型现在先输出唯一视觉锚点和构图，再决定 `selfie` 或 `text2img`，减少泛天气图、多主体拼贴和无依据自拍；链路测试、日志和扩展页摘要会显示视觉锚点/构图，方便区分“提示词设计跑偏”和“生图后端执行跑偏”。
- QQ 空间说说配图默认开启：`enable_qzone_generated_image_publish` 的代码默认值和配置页 schema 默认值改为 `true`，配置页说明补充“手动发说说未附图时会直接尝试生成配图”。
- 手动发说说接入自动配图：扩展页发布、聊天命令“陪伴 发说说”和 LLM 工具发布在未附图且有正文时，会调用 `_maybe_generate_qzone_publish_image(reason="manual_publish", force=True)` 生成配图；自动生活说说/情绪说说仍按原概率逻辑，不会被强制每次配图。
- QQ 空间配图策略调整为人物优先：配图提示词优先选择 `selfie`，让人物成为主角；即使没有明确自拍动机，也鼓励使用第一视角手部、侧脸、背影、镜中局部、肩颈半身等自然入镜方式，避免纯景物照片。没有参考图时外貌参考人格描述和公开状态，不再强制降级为纯生活碎片。
- 手动发布配图状态独立记录为 `manual_publish` 前缀，避免覆盖自动生活说说的配图排障状态。
